## Project Overview

A pure front-end (no build step) virtual billionaire shopping experience — single-page app with 8 celebrity characters, 60+ items across 9 categories, real-time wealth animation, a comedic loan system, and bill image generation.

- **No build tooling, no npm, no server** — open `index.html` directly in a browser.
- External runtime deps: `canvas-confetti` via CDN, Google Fonts (Orbitron + Noto Sans SC).

## Architecture (vanilla JS, no framework)

```
index.html              # DOM structure, CDN scripts, semantic sections
css/style.css           # All styling — CSS custom properties, responsive (768/420px breakpoints)
js/data.js              # Data layer: CHARACTERS, CATEGORIES, ITEMS, LOAN_CONFIG
js/app.js               # All logic: state, rendering, events, animations, canvas bill image
assets/products/        # ~50 product images (JPG), generated by the Python script
generate_images.py      # Batch image generator via Agnes-Image-2.1-Flash API
```

### Wealth Model (critical — easy to get wrong)

`state.wealth` tracks **remaining money after cart deductions**, not the character's initial wealth.

- When items are added: `state.wealth -= item.price` (animated via requestAnimationFrame)
- When items are removed: `state.wealth += itemTotal`
- `getBudget()` returns `Math.max(0, state.wealth)` — **NOT** `state.wealth - getCartTotal()`, because `state.wealth` is already reduced by the cart.
- Same rule applies everywhere "remaining" is displayed: use `state.wealth`, not `state.wealth - cartTotal`.

Breaking this rule causes double-counting (e.g. buying Nike → wealth drops to $85B, but `getBudget()` subtracts the cart again → shows $0 erroneously).

### Data Layer (`js/data.js`)

Four top-level constants:

- **`CHARACTERS`** — 8 billionaires with id, name, wealth, accent color, gradient theme. `imageUrl` optional; renders fallback emoji when absent.
- **`CATEGORIES`** — 9 tabs (food, daily, tech, fashion, travel, social, luxury, company, realestate). Each has only `id` and `name` (no `icon` field).
- **`ITEMS`** — 60+ products with id, category, price, emoji, optional `imageUrl`, `toast`, `charToasts`, and `unaffordableForAll` flag (Apple/Google).
  - `toast`: generic easter egg shown on add-to-cart
  - `charToasts`: character-specific overrides keyed by char `id`, `_default`, or `_qty_N` (shown when cart quantity reaches N)
  - `unaffordableForAll`: true disables purchase entirely (no loan option)
- **`LOAN_CONFIG`** — 10-stage progressive collateral system (left kidney → soul → rejection), 666% interest rate, comedic thresholds.

### App Logic (`js/app.js`)

**State** — a single `state` object (currentCharId, cart{}, wealth, totalLoan, loanCount, checkoutCompleted, loanedItems).

**Rendering** — imperative DOM manipulation via cached element refs (`dom.*`). Re-renders are full innerHTML replacements on the affected container. No virtual DOM or reactive system. XSS protection via `escapeHtml()` on all text content.

**Key modules:**

| Function Area | Key Functions | Notes |
|---|---|---|
| Wealth animation | `animateWealthTo()`, `updateWealthDisplay()` | easeOutCubic via requestAnimationFrame |
| Cart | `addToCart()`, `updateQuantity()`, `removeItem()`, `clearCart()` | Wealth deducted per-item-price, not cart-total |
| Toast | `showToast()`, `showToastForItem()` | 5-level priority: _qty_N → char-specific → _default → item.toast → fallback |
| Loan | `openLoanGeneral()`, `openLoan()`, `confirmLoan()`, `closeLoan()` | Combined collateral packages; `pendingCollaterals` stores original loan indices (not computed positions) |
| Checkout | `openCheckout()`, `confirmCheckout()`, `closeCheckout()` | Funny review text, confetti, canvas bill, receipt copy |
| Bill image | `generateBillImage()` | Canvas 600px; review text split by `<br>`, wrap at 36 chars, max 6 lines |
| Reset | `resetAll()` | Clears cart, loans, resets wealth to char initial |
| Init | `initApp()` | Sets event listeners on DOMContentLoaded; event delegation on productGrid and cartItems |

### Styling (`css/style.css`)

- CSS custom properties for theming (gold accent, glass morphism, dark space theme)
- Per-character background gradient set via `document.body.style.background`
- Component classes follow BEM-like naming (`.product-card`, `.cart-fab`, `.checkout-review`)
- Two responsive breakpoints: 768px (tablet) and 420px (small phone)
- Z-index layering: header 100, cart FAB 90, cart drawer 151, toast 180, modal overlay 200

### Loan System Architecture

- `pendingLoanItemId` tracks which item triggered the current loan dialog (null = general loan)
- `state.pendingCollaterals` array stores **original loan indices** (set in `openLoan()`, consumed in `confirmLoan()`)
- `openLoan()` calculates a combined collateral package: iterates from `state.loanCount` forward until total collateral value covers the price gap. Each needed collateral stores `{ index, name, value }` — the index is stored directly, not computed from position.
- `confirmLoan()` moves the ceiling check to AFTER `batchValue` calculation: the check uses `batchValue` (actual collateral amount added), not `loanAmount` (the price gap), preventing overshoot.
- Window: `openLoanGeneral()` is for direct cash loans; `openLoan(itemId)` is triggered by item purchase.

### Toast Priority System

`showToastForItem()` uses this chain (first non-empty wins):

1. `item.charToasts['_qty_' + currentQty]` — e.g. `_qty_10` fires on the 10th add
2. `item.charToasts[char.id]` — character-specific, e.g. `musk`, `jackma`
3. `item.charToasts._default` — fallback for all characters
4. `item.toast` — generic item toast
5. Category-based fallback (`food` → emoji + name, else → generic)

## Common Development Tasks

### Adding a new character
1. Add entry to `CHARACTERS` in `data.js`
2. Optionally add `imageUrl` pointing to `assets/products/` (portrait image)
3. Add character-specific toasts (`charToasts[charId]`) on relevant items

### Adding a new product
1. Add entry to `ITEMS` in `data.js` with id, name, category, price, emoji, desc
2. Optional: add `imageUrl`, `toast`, `charToasts`, `unaffordableForAll`
3. Add product image to `assets/products/` (can batch-generate via `generate_images.py`)

### Adding a quantity-based toast
Add `_qty_N` key to `item.charToasts` (e.g. `_qty_10: 'Custom message'`). The system automatically matches the current cart quantity — no code change needed.

### Generating product images
- `generate_images.py` sends prompts to Agnes-Image-2.1-Flash API, saves PNGs, and auto-updates `data.js` with local `imageUrl` paths
- **API key**: read from `AGNES_API_KEY` env var — set with `set AGNES_API_KEY=sk-...` before running. Do NOT hardcode keys in the script.

### Character switching
- `switchCharacter()` resets cart, loans, and wealth. Confirms before clearing non-empty cart.
- Sets per-character background gradient and accent border color.

### Common pitfalls
- **Wealth double-counting**: never compute `state.wealth - getCartTotal()` — `state.wealth` already reflects the deduction
- **Canvas text truncation**: the bill image review wraps at 36 chars / 6 lines max; long reviews are silently truncated
- **Z-index layering**: keep new modals at 200+, toasts at 180, cart drawer at 151 — stack in the right order
- **XSS**: always use `escapeHtml()` when rendering user-visible text into `innerHTML` (name, desc, tagline)
